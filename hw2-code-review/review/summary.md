# Итоговое ревью и приоритеты

## Краткое описание проекта и выбранной области анализа

**Проект**: HQL Builder - Java библиотека для упрощения создания HQL (Hibernate Query Language) запросов.

**Основная область анализа**: Архитектура кода, качество реализации, безопасность, тестирование и документация.

**Текущая версия**: 1.0.0-r2

**Цель библиотеки**: Предоставить удобный Builder API для создания HQL запросов с поддержкой domain-specific language (DSL) для типобезопасного построения путей свойств сущностей.

## Найденные проблемы, сгруппированные по типам

### 1. Архитектурные проблемы (Критичность: Высокая)

| Проблема | Описание | Сложность решения |
|----------|----------|------------------|
| Нарушение инкапсуляции | Многие классы имеют `private` поля с `protected` геттерами | Низкая |
| Повторяющаяся логика | Классы `AndExpression` и `OrExpression` имеют идентичную структуру | Низкая |
| Смешение ответственности | `CommonWhereExpression` выполняет слишком много функций | Средняя |
| Отсутствие разделения состояния | `HQLBuilder` хранит состояние и отвечает за обработку | Высокая |

### 2. Баги и потенциальные проблемы (Критичность: Критичная/Высокая)

| Проблема | Описание | Последствия |
|----------|----------|-------------|
| Мутация состояния в subquery | Прямая модификация `builder.getVariables()` | Конфликты переменных, непредсказуемое поведение |
| Отсутствие валидации | Нет проверок для UPDATE/DELETE запросов | Некорректные запросы |
| Неполная валидация | Отсутствует проверка null для многих параметров | NullPointerException |
| Отсутствие проверки на дубликаты | Можно добавить одну колонку несколько раз | Избыточные запросы |
| Непроверяемые null | Отсутствие проверок в `Expressions.java` | Потенциальные ошибки |

### 3. Проблемы читаемости и поддержки (Критичность: Средняя)

| Проблема | Описание | Сложность решения |
|----------|----------|------------------|
| Отсутствие JavaDoc | Ни один класс не имеет JavaDoc | Высокая (требует усилий) |
| Смешение логики и побочных эффектов | Методы инициализации модифицируют состояние | Средняя |
| Плохая обработка null | Отсутствие единообразной валидации | Низкая |
| Непонятные имена | Некоторые переменные неинформативны | Низкая |

### 4. Проблемы тестирования (Критичность: Высокая)

| Проблема | Описание | Последствия |
|----------|----------|-------------|
| Недостаточное покрытие | Около 30-40% покрытия | Риск регрессий |
| Отсутствие тестов исключений | Нет тестов на генерацию исключений | Непроверенная валидация |
| Отсутствие интеграционных тестов | Нет тестов с реальной ORM | Непроверенная интеграция |
| Отсутствие тестов производительности | Нет JMH тестов | Непроверенная производительность |
| Отсутствие тестов конкурентности | Нет тестов многопоточности | Потенциальные race conditions |

### 5. Проблемы документации (Критичность: Высокая)

| Проблема | Описание | Последствия |
|----------|----------|-------------|
| Отсутствие JavaDoc в коде | Ни один класс не документирован | Сложность использования |
| Плохая структура README | README труден для навигации | Потеря времени пользователями |
| Отсутствие расширяемой документации | Нет руководства по расширению | Сложность вклада в проект |
| Отсутствие диаграмм | Нет визуальных представлений архитектуры | Сложность понимания |

## Конкретные примеры кода

### Проблемный код #1: Мутация состояния

**Файл**: `CommonWhereExpression.java`

```java
protected String processVariable(Object input, HQLBuilder builder) {
    if (input instanceof HQLBuilder) {
        HQLBuilder incomingBuilder = (HQLBuilder) input;
        builder.getVariables().putAll(incomingBuilder.getVariables()); // ПРОБЛЕМА!
        // Прямая мутация состояния, дубликаты, непредсказуемость
    }
    // ...
}
```

**Рекомендуемое решение**:
```java
protected String processVariable(Object input, HQLBuilder builder) {
    if (input instanceof HQLBuilder) {
        HQLBuilder incomingBuilder = (HQLBuilder) input;
        String subqueryPrefix = "sub_" + System.nanoTime() + "_";
        incomingBuilder.getVariables().forEach((name, value) -> 
            builder.addVariable(value, subqueryPrefix + name));
        return "(" + incomingBuilder.build() + ")";
    }
    // ...
}
```

### Проблемный код #2: Отсутствие валидации

**Файл**: `ColumnExpression.java`

```java
public ColumnExpression(String path, ColumnExpressionType type) {
    super(ExpressionType.COLUMN);
    if (null == path || path.isEmpty()) throw new IllegalArgumentException("Path may not be empty.");
    if (null == type) throw new IllegalArgumentException("ColumnExpressionType may not be empty.");
    // ...
}
```

**Рекомендуемое решение**:
```java
public ColumnExpression(String path, ColumnExpressionType type) {
    super(ExpressionType.COLUMN);
    requireNotBlank(path, "path");
    requireNonNull(type, "type");
    // ...
}

private void requireNotBlank(String value, String name) {
    if (value == null || value.isEmpty()) {
        throw new IllegalArgumentException(name + " may not be empty");
    }
}

private void requireNonNull(Object value, String name) {
    if (value == null) {
        throw new IllegalArgumentException(name + " may not be null");
    }
}
```
