# HQL Builder - File Structure

## Структура проекта и назначение файлов

```
project-root/
├── memory-bank/          # Данный каталог с памяткой о проекте
│   ├── overview.md       # Обзор проекта и его назначение
│   ├── core-components.md # Основные компоненты API
│   ├── api-examples.md   # Примеры использования API
│   ├── important-notes.md # Важные замечания и ограничения
│   └── file-structure.md # Текущий файл - описание структуры
├── src/
│   ├── main/
│   │   ├── java/
│   │   │   └── org/
│   │   │       └── adaptms/
│   │   │           └── hqlbuilder/
│   │   │               ├── builder/      # Классы билдера запросов
│   │   │               ├── expression/   # Классы выражений
│   │   │               ├── property/     # Классы свойств и путей
│   │   │               └── util/         # Утилитные классы
│   │   └── resources/
│   │       └── (файлы ресурсов, если есть)
│   └── test/
│       └── java/
│           └── org/
│               └── adaptms/
│                   └── hqlbuilder/
│                       └── test/         # Тесты библиотеки
├── .idea/                # Конфигурация IntelliJ IDEA
├── .git/                 # Git репозиторий
├── pom.xml               # Конфигурация Maven
├── README.md             # Основная документация проекта
└── LICENSE               # Лицензия проекта
```

## Подробное описание каталогов

### memory-bank/

**Назначение**: Хранение документации и памятки о проекте для разработчиков

**Файлы**:
- `overview.md` - Общее описание проекта, его возможности и состояние
- `core-components.md` - Детальное описание основных компонентов API
- `api-examples.md` - Готовые примеры использования API
- `important-notes.md` - Важные замечания и ограничения
- `file-structure.md` - Текущий файл со структурой

**Формат**: Markdown (.md) файлы для удобного чтения и хранения информации

### src/main/java/org/adaptms/hqlbuilder/

**Назначение**: Основной исходный код библиотеки

**Подкаталоги**:

#### builder/

Классы, отвечающие за построение запросов:
- `HQLBuilder.java` - Основной класс билдера
- Методы для создания запросов разных типов (select, update, delete)

#### expression/

Классы выражений для WHERE-условий:
- `Expressions.java` - Статические методы для создания выражений
- `CommonWhereExpression.java` - Интерфейс для выражений
- Конкретные реализации выражений (eq, like, between и т.д.)

#### property/

Классы для работы со свойствами сущностей:
- `EntityPath.java` - Базовый класс для путей свойств
- `EntityPathFactory.java` (если есть) - Фабрика для создания путей
- Классы для DSL поддержки

#### util/

Утилитные классы:
- `QueryOrderDirection.java` - Перечисление направлений сортировки
- `ColumnExpressionType.java` - Перечисление типов колонок
- `JoinType.java` - Перечисление типов JOIN
- Вспомогательные классы и утилиты

### src/test/java/org/adaptms/hqlbuilder/test/

**Назначение**: Юнит-тесты библиотеки

**Структура**:
- Соответствует структуре src/main/java
- Тесты для каждого компонента
- Примеры использования в тестах

### .idea/

### README.md

## Работа со структурой

### Добавление нового компонента

1. Определить, в какой подкаталог `src/main/java/org/adaptms/hqlbuilder/` он попадет
2. Добавить исходный файл в соответствующий каталог
3. Добавить тесты в `src/test/java/` с соответствующей структурой
4. Обновить документацию в `memory-bank/`, если компонент касается публичного API

### Изменение API

1. Обновить документацию в `memory-bank/core-components.md`
2. Добавить примеры в `memory-bank/api-examples.md`
3. Обновить тесты
4. Обновить README.md при значительных изменениях

### Релизная подготовка

1. Проверить полноту документации в `memory-bank/`
2. Обновить версию в `pom.xml`
3. Создать тег в Git
4. Подготовить артефакт для публикации в Maven
