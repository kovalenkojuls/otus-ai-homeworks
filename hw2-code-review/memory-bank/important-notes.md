# HQL Builder - Important Notes

## Важные замечания при использовании

### Требования к сущностям

- **Обязательная аннотация `@Entity`**: Все сущности, участвующие в запросах, должны быть аннотированы как `@Entity`. Иначе билдер выбросит исключение.

### Особенности UPDATE и DELETE запросов

- **Без алиасов**: UPDATE и DELETE запросы не поддерживают алиасы. Используйте `root()` вместо `alias(String)` при построении путей.
- **Ограничения**: INSERT запросы не поддерживаются из-за ограничений HQL. Используйте ORM-функциональность Hibernate для вставки данных.

### Параметры и переменные

- **Автоматическое именование**: Параметры автоматически получают имена вида `:var_<alias><number>` (например, `:var_me0`, `:var_ae1`)
- **Маппинг параметров**: Используйте `getVariables()` для получения карты параметров и их установки в Hibernate Query
- **Безопасность**: Все параметры автоматически становятся привязанными переменными, что предотвращает SQL-инъекции

### JOIN-выражения

- **Типы JOIN'ов**: Все типы JOIN'ов (LEFT, RIGHT, INNER, FULL) должны быть явно указаны
- **With выражение**: Для JOIN используется выражение `with`, а не `on`. В HQL `with` используется для условий соединения.
- **Каскадные JOIN'ы**: Можно добавлять несколько JOIN'ов в одном запросе

### Выражения и их аргументы

- **EntityPath**: Большинство выражений принимают `EntityPath` как первый аргумент
- **Смешивание типов**: Можно смешивать `EntityPath`, `HQLBuilder` и обычные объекты в одном выражении
- **Подзапросы**: Подзапросы создаются как отдельные `HQLBuilder` объекты и встраиваются в основные выражения

### Производительность

- **Параметризация**: Всегда используйте параметризованные запросы через `getVariables()` для оптимальной производительности
- **Кэширование**: Стоит кэшировать часто используемые запросы и переиспользовать `HQLBuilder` с измененными параметрами
- **JOIN'ы**: Избегайте избыточных JOIN'ов, которые могут замедлить выполнение запросов

### Ограничения библиотеки

- **Неподдерживаемые типы запросов**: INSERT не поддерживается из-за ограничений HQL
- **Сложные выражения**: Некоторые сложные HQL-конструкции могут не поддерживаться
- **Порядок вызовов**: Порядок вызовов методов (column, where, join, orderBy и т.д.) может влиять на результат

### Отладка

- **Проверка HQL**: Используйте `build()` для получения готового запроса и логирования перед выполнением
- **Проверка параметров**: Используйте `getVariables()` для проверки всех параметров
- **Unit-тесты**: Пишите unit-тесты для каждого сложного запроса

### Обработка ошибок

- **Аннотации**: Проверка наличия `@Entity` происходит на этапе построения запроса
- **Пути**: Ошибки в путях свойств обнаруживаются на этапе выполнения HQL, если они не соответствуют реальной структуре сущности
- **Типы данных**: Убедитесь, что типы данных в выражениях соответствуют типам свойств сущностей
