# Архитектура приложения CookHub

CookHub реализует **микросервисную архитектуру**, где каждая функциональная область выделена в отдельный сервис, взаимодействующий с другими через асинхронные каналы (Kafka) или напрямую (HTTP).

## Общая схема

```
[Пользователь]
     │
     ├──▶ Web Application (Spring Boot + Thymeleaf)
     │       │
     │       ├──▶ PostgreSQL (данные, сессии, пользователи, рецепты, подписки)
     │       └──▶ Kafka (produce: registration emails)
     │
     ├──▶ Email Service (Spring Boot) ◀── Kafka (consume)
     │
     └──▶ Telegram Bot (Spring Boot)
```

## Компоненты и ответственность

### 1. CookHub Web Application (main)

**Порт**: 8080  
**Описание**: Центральный элемент, предоставляющий UI и обрабатывающий логику взаимодействия пользователя с системой.

#### Основные функции:
- **Аутентификация и авторизация**: Spring Security + Spring Session (хранение сессий в PostgreSQL).
- **Регистрация**:
    - Валидация форм (Hibernate Validator).
    - Отправка email-уведомления через Kafka.
    - Интеграция reCAPTCHA для защиты от ботов.
- **Рецепты**:
    - CRUD операции (Create, Read, Update, Delete).
    - Динамическая генерация форм для категорий из Enum.
    - Фильтрация по автору и категории.
    - Пагинация (для больших списков).
    - Загрузка и замена изображений (через `cookhub-uploadimages/`, если отдельный сервис).
- **Социальные функции**:
    - Подписки на авторов.
    - Лайки к рецептам.
- **Админ-панель**:
    - Доступ по роли `ADMIN`.
    - Управление пользователями.

#### Технологии:
- Java, Spring Boot, Spring Security, Spring Session, Spring Data JPA.
- База: PostgreSQL.
- UI: Thymeleaf, Bootstrap, JavaScript.

### 2. CookHub Email Service

**Порт**: (обычно 8081 или 8082, зависит от настройки)  
**Описание**: Отдельный микросервис, ответственный за отправку email-уведомлений. Декуплирует логику отправки из основного приложения.

#### Основные функции:
- **Консьюмер Kafka**: Подписывается на топик с данными для отправки email.
- **Генерация писем**: Использует шаблоны (например, HTML-шаблоны через Thymeleaf) для формирования содержимого письма (регистрация, подтверждение, уведомления).
- **Отправка писем**: Через SMTP (настраивается через переменные окружения).

#### Технологии:
- Java, Spring Boot.
- Apache Kafka (consumer).
- Spring Mail (вероятно, для SMTP).

### 3. CookHub Telegram Bot

**Порт**: Не имеет веб-интерфейса (обычно работает как "фоновая" служба)  
**Описание**: Бот в Telegram для предоставления функционала платформы без открытия браузера.

#### Основные функции:
- **Интеграция с Telegram API**: Обработка команд и сообщений пользователей.
- **Предоставление рецептов**: Основная функция — отправка случайных рецептов по категориям.
- **Краткий интерфейс**: Данные отправляются в виде текста или кнопок (в рамках Telegram API).

#### Технологии:
- Java, Spring Boot.
- Telegram Bots API (через библиотеку, например, `org.telegram:telegrambots`).
- Hibernate (для доступа к данным рецептов, возможно, через REST API Web Application или прямой доступ к БД — не указано).

### 4. Дополнительные компоненты (инфраструктура)

Описаны в Docker-конфигурации:

- **PostgreSQL**: Основная СУБД (содержит таблицы: `users`, `recipes`, `sessions`, `subscriptions`, `likes`, etc.).
- **pgAdmin**: Веб-интерфейс для управления PostgreSQL.
- **Zookeeper**: Координация для Kafka.
- **Kafka**: Брокер сообщений для асинхронного взаимодействия между микросервисами.
- **Kafka UI**: Визуализация и мониторинг Kafka (топики, сообщения, потребители).
- **Контейнер генерации топиков**: Автоматически создает необходимые топики при запуске (например, `registration-emails`).

## Потоки данных

1.  **Регистрация**:
    - Пользователь заполняет форму в `Web Application`.
    - `Web Application` сохраняет пользователя в БД.
    - `Web Application` отправляет сообщение в Kafka (топик `registration-emails`) с данными для email-уведомления.
    - `Email Service` получает сообщение из Kafka, генерирует HTML-письмо и отправляет его.
2.  **Просмотр рецептов**:
    - `Web Application` делает запросы к PostgreSQL для выборки рецептов, применения фильтров и пагинации.
3.  **Telegram Bot**:
    - Пользователь отправляет команду в Telegram.
    - `Telegram Bot` получает команду, формирует ответ (например, случайный рецепт) и отправляет его пользователю.
    - Взаимодействие с БД или Web Application для получения данных о рецептах.

## Принципы проектирования

-   **Концентрация ответственности**: Каждый микросервис делает одну вещь хорошо.
-   **Асинхронная коммуникация**: Использование Kafka для неблокирующих операций (email).
-   **Слабая связанность**: Микросервисы не зависят друг от друга напрямую, только через схему сообщений.
-   **Управление конфигурацией**: Через переменные окружения в `start.sh` и Docker.
-   **Декларативная часть UI**: Статическая генерация форм на основе Enum-категорий (упрощает добавление новых категорий).
